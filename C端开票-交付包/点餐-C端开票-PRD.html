点餐——C端开票

产品需求文档（PRD）  |  v1.0  |  2026-06-15

文档概述

项目内容
需求名称点餐——C端开票 / 本地生活客服工作台——发票工单管理
涉及系统点餐管理后台、供应商管理系统、本地生活客服工作台（Web管理后台）
需求背景平台合作供应商因利润较低等原因，拒绝为C端用户开具发票，导致用户消费后无法取得发票，存在税务合规风险及客诉风险。为保障用户权益、规避平台合规风险，平台决定以自身作为开票主体，统一为C端用户开具发票。
需求目的支持C端用户申请开票，由平台财务线下开票后上传并回传抖店；建立供应商开票额度管控机制，保障开票合规；完善供应商侧结算数据可查能力。
抖店API文档 📄 查看商家开票列表接口 📄 上传开票结果接口
原型链接 📋 点餐管理后台 📦 供应商管理系统 🎫 本地生活客服工作台

一、点餐管理后台

1.1 开票列表

功能名称：开票列表 —— 待开票 / 已开票 / 已作废 / 全部

数据源

  • 主数据全部来自抖店（抖音电商平台），系统定时拉取同步
  • 同步条件：订单状态字段仅保留【已完成】的订单
  • 同步优先级：【已完成】+【待开票】状态优先级最高
  • 待红冲订单不修改已有数据；操作日志回传并去重

显示逻辑

  • 顶部 Tab 切换：【待开票】/【已开票】/【已作废】/【全部】，待开票Tab 带数量角标
  • 待开票列表 排序：按照最晚开票时间正序排【距离越近越在前】
  • 已开票列表 排序：按照数据进入页面顺序，倒叙
  • 已作废列表 排序：按照数据进入页面顺序，倒叙
  • 全部页面 排序：按照数据申请时间倒叙
  • 待开票 页面：不按照时间显示，显示全部的未处理的数据【其他tab页的数据默认显示近30天】
  • 可折叠额度管控面板（详见 1.4）
  • 列表字段：
    • 【发票抬头】与【税号】同列展示，各带复制按钮
    • 个人票只显示名字，不显示税号
    • 最晚开票时间--数据源：接口拉取的

      若时间晚于当前时间，红字展示时间

    • 发票类型：接口拉取的
    • 上传时间/发票号：只有已开票/已作废状态展示
    • 作废时间/作废原因：只有已作废状态展示
  • 无数据时默认兜底显示：【-】
  • 导出按钮：导出字段除了操作列的字段【可以根据申请时间导出】
  • 一次导出数据不能超过10万，否则提示"最多能导出10万条"

交互逻辑

  • 不设批量开票，仅支持单条操作
  • 【待开票】状态下操作列按钮：【开票】
    • 点击触发开票弹窗
    • 弹窗内选择开票供应商（供应商 = 开票主体，非实际出餐方）
    • 开票供应商-数据源：开票出餐订单页面对应的平台订单号的供应商名称
    • 确认开票后：自动创建开票出餐记录（供应商 = 开票主体）
    • 财务线下完成开票 → 上传发票文件（PDF/OFD/XML/ZIP/RAR/7Z，≤10MB）
    • 上传成功后后台回传抖店，抖店返回上传结果
    • 发票抬头/税号校验：个人发票校验手机号格式，企业发票校验18位统一社会信用代码
  • 【已开票】状态下操作列按钮：【作废】
    • 已开票可作废，需填写作废原因
    • 详情弹窗：点击发票链接，在浏览器新窗口查看
    • 下载发票 按钮：点击按钮，在浏览器新窗口查看
  • 不设取消操作、不设驳回按钮
    • 原因：抖店已校验格式 + 同步导致死循环 + 超时扣分
  • 列表支持筛选：订单号、用户信息、发票类型、开票状态、申请时间范围
  • 支持定时刷新和手动刷新【拉取第三方数据】

状态机

开票管理所有页面的流转：订单号+店铺ID是唯一的

触发操作开票状态变化数据流转
用户申请开票(拉取数据)---待开票进入 待开票页面
未开票时，用户撤销申请待开票→已取消待处理页面→全部页面
撤销后，用户重新申请已取消→待开票全部页面→待处理页面
税务开票待开票→已开票待处理页面→已开票页面
已开票，用户要求开红字发票已开票→已作废已开票页面→已作废页面
未开票时，用户申请退款待开票→已取消待处理页面→全部页面
已开票，全退；若要求税务"再次上传"(红字)发票已开票→已作废已作废页面
上传红字后，用户重新申请已作废→待处理已作废页面→待处理页面

开票状态共 4 种：【待开票】→【已开票】→【已作废】；另有【已取消】。已开票不可回退到待开票，已取消仅由抖店同步驱动不可通过平台操作触发。

补充注意事项

  • 抖店待红冲工单线下处理，抖店对接配置不在页面上显示
  • 操作日志回传并去重，避免重复记录
  • 开票出餐记录中的供应商是开票主体，与实际出餐供应商分离

1.2 开票出餐列表

功能名称：开票出餐列表

数据源

  • 数据来源：原出餐订单列表的"出餐成功"状态的数据
  • 旧数据处理：配置供应商后，把出餐订单页面"出餐成功+有出餐时间"的数据同步到开票订单页面
  • 供应商 = 开票主体供应商（非实际出餐方）
  • 供应商-数据源：开票出餐订单页面配置的开票供应商
  • 支持手动添加：从系统出餐订单池搜索并添加

显示逻辑

  • 列表 24 列，含多选框，无操作列
  • 新增多选框：未选择时，点击"批量修改出餐供应商"，提示"至少选择一个订单"
  • 多选时，按照分页多选，也可按照筛选供应商+分页的单选
  • 无数据时默认兜底显示：【-】
  • 支持导出：导出列表字段与原出餐订单页的导出字段一致

交互逻辑

  • 支持手动添加记录：点击【添加】按钮弹出搜索弹窗，从系统出餐订单池搜索选择，确认后添加到列表中

补充注意事项

  • 开票出餐列表展示的是以开票供应商为主体的出餐记录

1.3 手动添加开票出餐订单

功能名称：手动添加开票出餐订单

交互逻辑

  • 1.输入渠道订单号 失焦后，搜索与渠道订单号相关的出餐订单，若有多个，显示在出餐订单的下拉选项里【多个时，下拉选项随机显示一个】；若没有，在渠道订单号输入框下面红色小字显示"该订单没有关联到出餐订单，请重新输入"
  • 2.点击 确定，校验必填项；以及选择的出餐订单是否已经在开票出餐订单列表内，若存在，提示"该订单已经在开票出餐订单列表，请重新输入"
  • 保存之后，提示'保存成功'，添加对应的开票出餐订单，且除了渠道订单号、出餐订单、供应商名称外，其他的字段跟原出餐订单相同出餐订单的字段取值是一样的
  • 添加开票出餐订单时，同步在对应的供应商系统里增加出餐订单信息，若有结算记录，增加结算记录
  • 注意：可以不增加对应的接单记录

1.4 开票额度管控面板

功能名称：开票额度管控面板

数据源

当前供应商 下拉选项-数据源：当前页面已所有的供应商名称

显示逻辑

  • 可折叠面板，放在开票列表顶部
  • 按供应商展示：额度类型（月度/季度/年度）+ 已用额度 + 剩余额度 + 使用比例
    • 额度类型（月度/季度/年度）--数据源：配置弹窗配置的额度周期
    • 免税额度--数据源：弹窗配置的
    • 已开票金额--数据源：该页面所有已开票的数据的开票金额之和
    • 剩余额度--数据源：免税额度-已开票金额
  • 使用比例超过阈值时高亮红色警告

交互逻辑

  • 修改配置：点击弹出弹窗

    弹窗内容：

    1. 开票供应商--数据源：开票出餐订单列表已有的供应商名称
    2. 额度周期：枚举值【月度、季度、年度】
    3. 免税额度--输入逻辑：0-9999999，两位小数
    4. 告警阈值--输入逻辑：0-100，两位小数
    5. 告警接收人--输入逻辑：飞书企业绑定手机号，多个逗号隔开
    6. 是否启用飞书提醒 按钮：默认：开启；关闭不进行飞书提醒
    7. 保存 按钮：点击，提示"保存成功"；数据回显页面；若启用飞书提醒，达到设置的阈值，进行飞书提醒
      1. 开票额度监控面板下面的告警阈值为弹窗回显；启用飞书提醒时，显示"飞书通知--已启用"；反之，提示"飞书通知--已禁用"
  • 阈值触发飞书告警通知
  • 面板支持展开/折叠

补充注意事项

  • 飞书提醒文案：
    • 配置的供应商名称的 "填写的额度周期"免税额度为"填写的免税额度"元；剩余额度为"填写的免税额-统计的开票金额 的值"。已超出设置的告警阈值 设置的百分比。请及时更换开票供应商
      @填写手机号的企业名称
  • 飞书提醒频率：每个小时提醒一次

1.5 抖店数据同步

功能名称：抖店数据同步

数据源

  • 抖店平台开票申请数据，定时拉取【10分钟/次】
  • 可以手动触发【上传失败时，可手动拉取】

交互逻辑

  • 同步时机：定时任务自动执行
  • 同步条件：订单状态为【已完成】
  • 优先级：【已完成】+【待开票】状态优先同步
  • 待红冲订单：不修改已有数据
  • 操作日志：回传并去重

二、财务管理模块

2.1 结算记录

功能名称：结算记录列表

数据源

  • 页面 新增 开票供应商的的未出餐扣减流水及订单完成结算流水
    1. 订单在开票出餐订单页面并进行售后，并且扣回结算金额----增加未出餐扣减流水
    2. 订单在开票出餐订单页面且结算状态=已结算----订单完成结算流水

2.2 资金账户记录

功能名称：资金账户记录列表

数据源

  • 页面 新增 开票供应商的的未出餐扣减流水及订单完成结算流水
    1. 订单在开票出餐订单页面并进行售后，并且扣回结算金额----增加未出餐扣减流水
    2. 订单在开票出餐订单页面且结算状态=已结算----订单完成结算流水

2.3 退款弹窗

功能名称：销售订单退款

交互逻辑

  • 确认 按钮：
  • 新增逻辑：
    • 点击确定时，给该订单关联的所有供应商都进行退款（退款金额相同），并添加"未出餐扣减"类型的流水记录【包括开票订单的供应商】

四、通用规则

4.1 空数据统一规则

所有模块无数据、无记录、无内容时，默认兜底显示：【-】

4.2 名称匹配规则

通过 ID / 单号 / 匹配对应名称信息

4.3 操作记录规则

操作类记录固定展示：操作人名称 + 对应操作时间；多条记录按时间倒序展示。

4.4 上传/操作结果文案规则

  • 失败：固定展示【上传失败：失败原因】，多条记录取最后一次结果

4.5 列表与数据刷新规则

支持定时刷新、手动刷新；刷新时保留历史操作记录，不清空已产生的业务数据。
